Quick Start

Getting Started

  1. Read the Documentation
    • Branch Strategy - Understand our branching model
    • Contributing Guidelines - How to submit PRs
    • README - Project overview
  2. Fork and Clone 
    git clone git@github.com:YOUR-USERNAME/OpenMOVR.github.io.git
cd OpenMOVR.github.io
  3. Run Locally 
    ./launch.sh
# Or: python3 -m http.server 8000
# Visit: http://localhost:8000
  4. Create a Feature Branch 
    git checkout -b feature/your-feature-name

Branch Strategy

Main Branches

  • main - Production website (homepage, docs, GSoC, pilot)
  • viewer-apps - MOVR Viewer applications (research tools)

Development Workflow

For website changes:

git checkout main
git pull origin main
git checkout -b feature/your-feature
# Make changes
git commit -m "component: Description"
git push origin feature/your-feature
# Open PR to main

For MOVR Viewer changes:

git checkout viewer-apps
git pull origin viewer-apps
git checkout -b feature/viewer-enhancement
# Make changes
git commit -m "viewer: Description"
git push origin feature/viewer-enhancement
# Open PR to viewer-apps

Architecture

Tech Stack

  • Frontend: Vanilla HTML5, CSS3, JavaScript (no frameworks)
  • Hosting: GitHub Pages (static site)
  • Analytics: Google Analytics
  • Forms: Power Automate integration (pilot enrollment)

Design System

CSS variables defined in css/main.css:

  • Colors: MDA brand colors (blue, gold, red, green, purple)
  • Typography: Inter (primary), Playfair Display (secondary)
  • Spacing: Scale from xs to 3xl
  • Components: Cards, buttons, forms, navigation

Configuration

Site configuration is centralized in js/config.js:

  • Navigation structure
  • Team information
  • Legacy stats
  • Timeline and roadmap
  • Asset paths

Common Tasks

Adding a New Page

  1. Create HTML file in appropriate directory
  2. Include header component using <div id="header-placeholder"></div>
  3. Add <script src="js/header-loader.js"></script> in <head>
  4. Update navigation in js/config.js if needed
  5. Test locally

Updating Statistics

Edit js/config.js and update the legacy object:

legacy: {
    participantsTrusted: 5895,
    encountersLogged: 20152,
    sitesCollaborating: 63,
    yearsLearning: 12
}

Modifying Styles

  • Global styles: Edit css/main.css
  • Documentation styles: Edit docs/css/docs.css
  • Pilot styles: Edit pilot/style.css

Code Standards

HTML

  • Use semantic HTML5 elements
  • Include ARIA labels for accessibility
  • 2-space indentation
  • Meaningful class names

CSS

  • Use CSS variables for consistency
  • Mobile-first responsive design
  • BEM-like naming convention
  • Comment complex styles

JavaScript

  • Vanilla JavaScript (no jQuery, React, etc.)
  • Use const and let (not var)
  • Comment complex logic
  • Handle errors gracefully
  • Test cross-browser if possible

Commit Messages

component: Short description

Longer explanation if needed.
Explain WHY, not WHAT (code shows what).

Example prefixes: pilot:, viewer:, docs:, gsoc:, home:, config:, style:, fix:, chore:
